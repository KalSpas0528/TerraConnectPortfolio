Project Title & Description

TerraConnect: Global Interactive News Map
An educational platform combining an interactive Leaflet.js world map, on‑click Wikipedia summaries, a Country Guessing Game, side‑by‑side comparisons, and (in progress) live news via RSS feeds.

Research & Planning Documentation

To ensure performance, accessibility, and maintainability, I evaluated multiple approaches before settling on the current stack:

  • Mapping Library: Chose Leaflet.js for its lightweight bundle, extensive plugin ecosystem, and zero‑cost licensing. Compared with Mapbox GL (better styling but pay‑per‑use) and Google Maps (heavy SDK, usage quotas).
  • News Integration:
    • News APIs – Comprehensive articles but cost‑prohibitive for free tiers.
    • Web Scraping – Flexible but risks rate‑limits and latency spikes.
    • RSS Feeds – Lightweight XML feeds from major outlets; chosen for prototype speed and reliability.
  • Wireframing & UX Flow: Created detailed wireframes in Figma, mapping out user journeys: country click → Wikipedia popup → optional news pane.
  • Performance Planning: Decided on debounced search and minimal DOM updates to keep map fluid, even on slower connections.

Final Product Description

The final feature to be added is the news feature. When you click on a country, you get live news from that country. Once I add that aspect, my final prodcut product will be done. A Website where you get to become educated about the globe, learning politics, geography, current events, and geopolitics.

Target Audience Explanation

TerraConnect is designed for students and lifelong learners curious about geography, international politics, and global affairs. It solves the problem of fragmented learning resources—combining maps, Wikipedia summaries, and current news into a single interactive platform. Ideal for both classroom educators and independent learners seeking an engaging, unified way to explore the world.

Development Challenges

While building the final news feature, I encountered major technical roadblocks. Integrating live news feeds using an RSS parser or News API turned out to be much more difficult than expected — not only were the API results inconsistent, but connecting them smoothly into the UI caused my map and homepage to sometimes break or “freeze” on reload. Debugging these asynchronous data flows has been extremely tedious, and revealed weaknesses in my original architecture that weren't obvious until now. These setbacks, though frustrating, highlighted how much I've grown since the start of the project: early on I focused mainly on visuals, but now I’m dealing with real-world API handling, error resilience, and UI stability at scale.
